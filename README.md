# A股量价因子研究与单因子评估框架


> 关键词：**A股日频因子研究 / 横截面 IC 分析 / 量价因子挖掘  / 研究框架搭建**

---

## 项目简介

这是一个围绕 **A股日频量价因子研究** 搭建的个人量化探索项目。  
项目的核心目标不是“堆很多因子”，而是尽量按照私募买方研究的思路，把一个单因子从 **想法提出 → 公式实现 → 全市场回测 → IC/分组诊断 → 失败复盘 → 因子改造** 这一整套流程跑通。

在这个项目里，我主要完成了三件事：

1. **搭建了一个可复用的单因子研究框架**  
   支持单股票时序因子计算、全市场横截面面板拼接、未来收益对齐、去极值标准化、IC/ICIR 评估、分组收益分析和多空诊断。

2. **系统性构造并测试了一批量价因子**  
   重点围绕成交量、VWAP（根据1min K线计算所得）、波动、动量、反转、资金流、跳空、分布偏度等信息，形成了一套自己的因子库与因子优化思路。

3. **把“研究诊断”作为核心环节，而不只是回测数值**  
   项目中不只看 Mean IC，而是重点分析：
   - 因子是否单调  
   - 是否存在“甜蜜区”  
   - G10 是否是毒药桶  
   - 是否存在 regime 切换  
   - 是否有明显的实现问题 / 数据口径问题 / 极端值污染

这也是我理解的量化研究实习中最重要的能力之一：  
**不是只会 vibe coding，还能能对图和结果形成有逻辑的研究判断。**

## 我想解决的问题

在很多初学量化项目里，研究流程常常停留在：

- 先写一个因子
- 再算一个 IC
- IC 看起来不错就算结束

但在真实研究里，这样远远不够。  
一个因子即使 IC 为正，也可能：

- 分组不单调
- 极端桶（G10）反而最差
- 多空收益完全不匹配
- 只在个别年份有效
- 被极端值、停牌、低流动性股票污染
- 甚至存在未来函数 / 对齐错误

所以这个项目更关注的问题是：

> **如何建立一个“能发现问题、能解释问题、能迭代因子”的研究流程？**

## 项目亮点

### 1. 单因子研究流程完整
项目不是单纯的“因子生成器”，而是包含了一条完整研究链路：

- 单股票日线数据读取与清洗
- 因子类封装（`FactorBase`）
- 全市场并发计算因子
- 多持有期未来收益对齐（T+1 口径）
- 截面 MAD 去极值 + Z-score 标准化
- 日度 IC / ICIR / WinRate / T-stat
- 十分组收益分析
- Long-Short 利差分析
- 因子秩自相关（换手代理）
- 自动输出图表与研究报告

### 2. 因子不只“做出来”，还要“看明白”
我在项目里对大量因子做了逐个分析，不只判断“好/坏”，还会进一步区分：

- **单调因子**：高分位更强
- **反向因子**：高值反而更差
- **非线性因子**：分组收益中，中间组最好，极端组很弱甚至垃圾
- **风险过滤因子**：高值更多反映拥挤、事件态、不可交易风险
- **regime 因子**：某些年份有效，某些年份失效

这让我对“因子有效”这件事有了更细的理解，也逐步学会避免“只看一个 IC 就下结论”的研究误区。

### 3. 做了因子改造，而不是止步于原始结果
针对表现较好的因子（例如部分 VWAP / 价量相关 / 跳空确认 / 组合因子），我进一步做了优化尝试，例如：

- 极端值截尾 / 地板处理
- `ts_rank(vol)` 改为 `vol/adv`
- 用 `avg(Ga..Gb)-avg(Gc..Gd)` 替代单桶 `G10-G1`
- 从“极端桶”转为“甜蜜区”构造
- 把简单等权组合改为带权组合
- 对部分信号加入“中间提拔”的距离映射

这一部分让我对“研究不是做一个公式，而是做一套更稳定的信号表达”体会很深。

### 4. 尝试把算子抽象成通用库
在项目迭代中，我开始把因子计算中反复出现的操作抽象成通用算子库（`factor_operator.py`），例如：

- rolling mean / sum / std / argmax / argmin
- ts_rank / decay_linear / corr
- vwap / adv / delta / signed_power
- 收益率计算、基础时序工具等

这样做一方面让因子公式更清晰，另一方面也更接近工业化研究代码的组织方式。

## 研究方法概述

### 1. 数据与收益口径
- 频率：**A股日频**
- 研究对象：单股票日线 OHLCV / amount
- 收益定义：  
  使用 **T+1 收盘买入、T+1+h 收盘卖出** 的未来收益率  
  这样做主要是为了尽量避免把当日收盘后才完整知道的信息，直接用于当日交易。

### 2. 因子评估指标
项目里主要关注以下指标：

- **Mean IC**
- **ICIR**
- **Win Rate**
- **T-stat**
- **多 horizon 的累计 IC 曲线**
- **十分组累计收益**
- **Long-Short 利差**
- **Rank Autocorrelation**（换手代理）

### 3. 结果解读方式
项目中我形成了一个固定的读图框架：

#### 累计 IC
看方向、稳定性、最佳持有期、是否存在风格切换。

#### 分组累计收益
看是否单调、是否存在甜蜜区、极端桶是否为毒药桶、是否存在飞天桶/贴地桶。

#### 多空利差
看当前构造是否匹配分组图，是否存在 regime，是否有异常台阶或样本塌缩。

## 一些有代表性的研究结论

以下是项目中一些比较有代表性的体会：

### 1. “IC 为正”不代表“G10 就该做多”
这是这个项目给我最强的一条反馈。  
很多因子虽然累计 IC 为正，但分组图显示：

- 真正的 alpha 在 **G5–G8**
- **G10 反而是毒药桶**

这意味着：  
**因子有排序能力 ≠ 极端高值就是最佳多头。**

### 2. 大量量价因子是“甜蜜区因子”
项目里很多因子都表现出这种结构：

- 极端低值差
- 极端高值也差
- 只有中间偏高区间最好

这让我开始把因子设计从“极端值排序”转向“中间状态表达”。

### 3. 很多“好看结果”其实是异常值/口径问题
当看到这些图形时，我会优先怀疑实现而不是立刻相信：

- 单桶飞天
- 全体贴地
- 多空台阶跳变
- IC 和收益端完全冲突
- 极端高强度长期单边

这也让我意识到，量化研究里“排障能力”和“怀疑精神”非常重要。

### 4. 某些因子更适合作为风控因子而不是 alpha
例如：
- 高波动 × 高放量
- 成交量 CV
- 偏度/高阶矩
- 冲击-量能相关

它们往往更适合做：
- 过滤极端风险桶
- 识别事件态
- 做拥挤/不可交易风险约束

而不是直接做多空 alpha。

## 如何运行

### 1. 定义或选择一个因子
在 `factor.py` 中新增一个 `FactorX(FactorBase)` 类，并实现 `formula()`。

### 2. 运行单因子评估
使用 `factor_ic_analysis.py` 对指定因子进行全市场评估。

项目核心流程包括：
- 并发读取股票 parquet
- 计算时序因子
- 构建横截面面板
- 对齐未来收益
- 做截面标准化
- 输出 IC / 分组 / 多空等结果

### 3. 查看输出
输出通常包括：
- 结果图
- 指标表
- 诊断信息
- Markdown 报告

> 由于本项目主要用于研究展示，具体路径和数据配置可根据本地环境调整。
